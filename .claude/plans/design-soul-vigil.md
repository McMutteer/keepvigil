# Vigil — Design Soul (Phase 1)

**Date:** 2026-03-17
**Status:** Approved

## Visual Tone (one sentence)
A professional terminal at 3am — everything is clear, everything is quiet, the only light is amber and it tells you exactly what you need to know.

## Personality → Design Translation

| Trait | Visual Primitive |
|-------|-----------------|
| **Silent** | Generous whitespace. No popups, chat widgets, autoplay. Content breathes. Sections separated by air, not borders. |
| **Reliable** | Strict spacing grid. Perfect alignment. No decorative randomness. Lines are straight, radii consistent. The page feels constructed, not decorated. |
| **Nocturnal** | Dark theme mandatory. Backgrounds in #0A-#12 range. Colors live in penumbra — deep blues, cold grays. A single warm light: amber. |
| **Precise** | Monospace for code and data. Mathematical spacing (4px base). Sharp corners on code blocks. Exact values, not approximations. "82/100" not "high confidence." |
| **Trustworthy** | Real evidence, not illustrations. Show actual Vigil output — the PR comment as it appears on GitHub. Data, not promises. Code blocks, not marketing speak. |

## Must-Haves (from personality + conversation)
- Dark mode only — no light mode toggle
- Real product output in the hero (the PR comment with score)
- Code blocks as design pieces (config, output, signals)
- Monospace accent font throughout
- Amber (#e8a820) as the only warm color — everything else is cool
- Evidence-first content strategy — every claim backed by an example
- Generous vertical rhythm — sections breathe

## Must-Nots (from "What We Are Not" + boundaries)
- No generic SaaS aesthetic (pastel gradients, character illustrations, aggressive CTAs)
- No "trusted by 500+ companies" social proof (doesn't exist yet)
- No video hero or autoplay media
- No chat widgets, popups, newsletter signup forms
- No competitor comparison table
- No dashboard screenshots (no dashboard exists)
- No light mode — this product lives at night

## Content Strategy
Evidence-first. Each claim is backed by a real example:
- "Vigil gives you a confidence score" → here's the actual score on a real PR
- "It scans for credentials" → here's what it looks like when it finds one
- "82/100" → here's exactly how each signal contributes
- The developer who visits doesn't want to be sold — they want to see what it does and decide alone

## Mood References
1. **Linear.app** — respect for whitespace, typography as protagonist, darkness as elegance. Borrow: information density without feeling heavy.
2. **Vercel.com** — the demo as hero, product shown working. Borrow: the hero IS the product, not a description of the product.
3. **Resend.dev** — developer tool nocturnal aesthetic, code blocks as design pieces, amber/gold as warm accent in cold darkness. Borrow: the tone that says "this is for you, developer" without saying it.
